# Security policy

## Supported versions

Only the latest published version of each `@rooted/*` package receives security fixes.

## Reporting a vulnerability

**Please do not open a public GitHub issue for security vulnerabilities.**

Use the GitHub private security advisory instead:

👉 [Report a vulnerability](https://github.com/Marvin-Brouwer/rooted/security/advisories/new)

You will receive a response within **5 business days**. If the issue is confirmed
we will work on a fix and coordinate a disclosure date with you before anything
is made public.

## Disclosure policy

Once a fix is released we will publish a GitHub Security Advisory with full
details, credit the reporter (unless they prefer to remain anonymous), and
release a patch version of the affected package(s).
