# rooted

> A TypeScript framework for web apps. Rooted in custom elements, typed end-to-end, batteries included.

Rooted is a framework for building web applications on top of native [custom elements](https://developer.mozilla.org/en-US/docs/Web/API/Web_components/Using_custom_elements). \
It gives you a typed component model, a typed router with per-route SEO, and a vertical-slice application layout. There is no virtual DOM, no signals, and no compiler step beyond standard TypeScript. Code reads top-to-bottom and does what it says.

```ts
import styles from './greeting.css'
import { component } from '@rooted/components'

export const Greeting = component({
  name: 'greeting',
  styles,
  onMount({ append, element }) {
    append(
      element('p', {
        textContent: 'Hello from rooted!',
      })
    )
  },
})
```

## Installation

Install the pieces you actually use.

```sh
# Core: components and app bootstrap
pnpm add @rooted/components

# Client-side router (also pulls in the Vite plugin for route discovery)
pnpm add @rooted/router

# Shared state, web storage helpers, and DX/SEO tooling
pnpm add @rooted/store @rooted/storage
pnpm add -D @rooted/application
```


## Quick start

### 1. Write a component

```ts
// src/counter.mts
import styles from './counter.css'
import { component } from '@rooted/components'

export const Counter = component({
  name: 'counter',
  styles,
  onMount({ append, element }) {
    let count = 0

    const label = append(
      element('span', {
        textContent: '0',
      })
    )

    append(
      element('button', {
        textContent: 'Increment',
        on: {
          click() {
            count++
            label.textContent = String(count)
          },
        },
      })
    )
  },
})
```

### 2. Bootstrap the app

```ts
// src/main.mts
import { application } from '@rooted/components/application'
import { component } from '@rooted/components'
import { Counter } from './counter.mts'

const Application = component({
  name: 'app',
  onMount({ append, element }) {
    append(
      element('h1', {
        textContent: 'My App',
      })
    )
    append(Counter)
  },
})

application(Application) // mounts on #app by default
```

### 3. Add routing (optional)

Routes live next to the components they render. Each feature folder owns a `_routes.mts` file, and the Vite plugin discovers them at build time.

```ts
// src/articles/_routes.mts
import { route, token } from '@rooted/router'
import { Article } from './article.mts'

export const ArticleRoute = route`/articles/${token('id', Number)}/`(Article)
```

```ts
// src/main.mts
import { application } from '@rooted/components/application'
import { router } from '@rooted/router/application'
import { appRoutes } from './_routes.g.mts' // generated by the Vite plugin

const Router = router({
  home:     HomeComponent,
  notFound: NotFoundComponent,
  ...appRoutes,
})

application(
  component({
    name: 'app',
    onMount({ append }) {
      append(Router)
    },
  })
)
```

See the [`recipe-book` example](./examples/recipe-book) for a full vertical-slice app.


## Packages

| Package | What's in it |
|---------|--------------|
| [`@rooted/components`](./packages/components) | The `component()` factory, the mount-context API, and the app bootstrap. |
| [`@rooted/router`](./packages/router) | Typed routes, gates, navigation helpers, and the Vite route-discovery plugin. |
| [`@rooted/localization`](./packages/localization) | URL-based localization: locale route token, overlay dictionaries, and hreflang tooling. |
| [`@rooted/elements`](./packages/elements) | Low-level DOM helpers used by components. Useful when you build elements without a component shell. |
| [`@rooted/events`](./packages/events) | Typed event helpers and global error utilities. |
| [`@rooted/store`](./packages/store) | A small synchronous store with `update` and `change` events. Not reactive. |
| [`@rooted/storage`](./packages/storage) | Typed wrappers around `localStorage`, `sessionStorage`, and cookies. |
| [`@rooted/application`](./packages/application) | Build-time configuration and SEO plugins (sitemap, llms.txt, per-route meta). |
| [`@rooted/util`](./packages/util) | Shared utilities (seeded IDs, client-side checks, dev helpers). |

## Development

```sh
# Clone and install
git clone https://github.com/Marvin-Brouwer/rooted.git
cd rooted
pnpm install

# Run all packages in watch mode and start the example app
pnpm dev

# Run tests
pnpm test

# Run tests in watch mode
pnpm test:watch

# Lint (auto-fix)
pnpm lint

# Build all packages
pnpm build
```
